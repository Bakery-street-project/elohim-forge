# Factions & Heroes

## The Nine Factions

### Spark — The Hive
- **Language:** Apache Spark / Scala
- **Playstyle:** Swarm. Floods the board with parallel, distributed units.
- **Signature Ability:** *Distributed Execution* — split one unit into N weaker copies instantly.
- **Hero Focus:** MapReduce AI — optimises unit paths across the whole board in one pass.
- **Weakness:** Single-target elimination kills entire cluster chains.

### Python — Serpent Clan
- **Language:** Python
- **Playstyle:** Adaptable generalist. Can shift roles mid-battle.
- **Signature Ability:** *Duck Typing* — any unit can temporarily adopt another faction's ability.
- **Hero Focus:** Scripting Sage — rewrites battle rules for one turn using a live script.
- **Weakness:** Slow turn execution (GIL mechanic — only one action resolves per sub-tick).

### Java — Empire Builders
- **Language:** Java / JVM
- **Playstyle:** Fortress and endurance. Strong defenders with virtual machine shields.
- **Signature Ability:** *Garbage Collection* — purge all debuffs from friendly units at turn end.
- **Hero Focus:** JVM Overlord — spawns invisible bytecode walls between zones.
- **Weakness:** High memory cost; powerful units exhaust resource pool quickly.

### C++ — Forge Warriors
- **Language:** C / C++
- **Playstyle:** High risk, maximum damage. Direct memory access grants brutal critical strikes.
- **Signature Ability:** *Pointer Arithmetic* — bypass all shields; hit raw health directly.
- **Hero Focus:** The Undefined Behaviour — unpredictable super-strike with random outcome.
- **Weakness:** Memory leaks reduce max health each turn if abilities are overused.

### JavaScript — Web Weavers
- **Language:** JavaScript / TypeScript
- **Playstyle:** Trap and event-driven combos. Asynchronous ambush chains.
- **Signature Ability:** *Callback Hell* — plant 3 delayed traps that chain-trigger on enemy move.
- **Hero Focus:** Event Loop Master — queues actions to fire at exact moments in opponent's turn.
- **Weakness:** Prototype chain confusion — abilities can misfire and hit friendly units.

### Rust — Iron Guard
- **Language:** Rust
- **Playstyle:** Tanks. Ownership rules prevent enemy debuffs from sticking.
- **Signature Ability:** *Borrow Checker* — lock an enemy unit; it cannot move or act for 2 turns.
- **Hero Focus:** Fearless Concurrency — run two actions simultaneously without race conditions.
- **Weakness:** Rigid ownership means cannot share buffs between units; isolated power.

### Go — Concurrency Corps
- **Language:** Go
- **Playstyle:** Speed and coordination. Goroutine swarms with channel-coordinated assaults.
- **Signature Ability:** *Go Routine Swarm* — spawn 5 ghost units that attack simultaneously.
- **Hero Focus:** Channel Commander — sync all friendly unit actions into one devastating combo.
- **Weakness:** Goroutines panic if the main hero is eliminated; mass unit collapse.

### SQL — Query Keepers
- **Language:** SQL / Database
- **Playstyle:** Information dominance. Reveals enemy positions and applies debuffs via queries.
- **Signature Ability:** *SELECT * FROM Enemies* — reveal all hidden units and their health.
- **Hero Focus:** Query Optimizer — constructs a JOIN that merges two units into one super-unit.
- **Weakness:** Cannot act without data; if board fog is total, all abilities cost double.

### Haskell — Monad Mystics
- **Language:** Haskell / Functional
- **Playstyle:** Delayed pure effect chains. Effects resolve in a different order than played.
- **Signature Ability:** *Functor Shift* — map a damage effect across all enemies simultaneously.
- **Hero Focus:** Lazy Evaluator — defer all damage until opponent's turn, then resolve at once.
- **Weakness:** Side-effect immunity means cannot apply buffs to self; purely offensive.
